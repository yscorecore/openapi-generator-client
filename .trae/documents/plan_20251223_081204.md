# 将项目改造为Monorepo风格的计划

## 1. 改造根目录package.json
- 添加`workspaces`配置，指定examples目录下的子项目
- 调整项目名称和描述

## 2. 在examples目录下创建子项目package.json
- 为client、global、global2、global3四个子目录创建各自的package.json文件
- 每个子项目将有自己的名称、版本和依赖

## 3. 调整脚本配置
- 将原有的生成脚本保留在根目录
- 添加子项目的构建、测试脚本

## 4. 调整依赖关系
- 确保共享依赖在根目录管理
- 子项目只包含自己特有的依赖

## 具体步骤
1. 修改根目录package.json，添加workspaces配置
2. 在examples/client创建package.json
3. 在examples/global创建package.json
4. 在examples/global2创建package.json
5. 在examples/global3创建package.json
6. 验证npm install是否正常工作
7. 测试各个生成脚本是否正常运行
